# 优惠券叠加计算引擎


## 获取工程

  git clone git@github.com:LeonGuo9/coupon-engine.git
  cd coupon-engine


## 背景

你接手了一个电商平台优惠券计算模块的代码。前任开发者写了一个可以运行的版本，但代码质量堪忧，且存在若干 bug。

你需要在 1 小时内，借助 AI CLI 编程工具完成以下任务。最终以 mvn test 全部通过作为基本验收标准。

运行方式：
  mvn test                                          # 运行所有测试
  mvn test -Dtest=CouponCalculatorAcceptanceTest     # 运行验收测试


========================================
  业务规则
========================================

--- 优惠券类型（4种） ---

  类型        枚举值                   说明                                               示例
  满减券      THRESHOLD_REDUCTION      订单金额满X元减Y元                                 满200减30
  折扣券      DISCOUNT                 订单金额打N折（value为折扣率，如0.8表示8折）        8折
  立减券      FIXED_REDUCTION          直接减固定金额                                     立减15元
  满赠券      THRESHOLD_GIFT           订单满X元赠送礼品（不影响金额，但需在结果中记录）   满300赠礼品

--- 叠加规则 ---

  1. 不同类型可叠加：折扣券、满减券、立减券可以同时使用
  2. 计算顺序：折扣 -> 满减 -> 立减（先折后减，对用户最优）
  3. 同类型互斥：同一类型的多张券只能使用一张，选取规则如下：
     - 先按来源优先级筛选：MEMBER（会员专属） > CAMPAIGN（活动券） > GENERAL（通用券）
     - 同来源的多张券，取优惠力度最大的那张
  4. 门槛检查：有门槛的券（threshold > 0）需要在「当前金额」满足门槛时才能使用
  5. 最低金额保护：最终金额不得低于 0.01 元
  6. 过期券过滤：已过期的优惠券应被忽略，不参与计算

--- 品类券 ---

部分优惠券限定适用品类（通过 applicableCategories 字段指定）：

  - 品类券的门槛检查和优惠计算，仅基于匹配品类的商品金额
  - applicableCategories 为 null 或空列表表示全品类通用
  - 品类券的优惠金额从订单总价中扣除

--- 冲突组 ---

部分券标记了相同的冲突组（conflictGroup 字段）：

  - 同一冲突组内的券即使类型不同也只能使用一张
  - 冲突组内应选取对当前订单优惠力度最大的券
  - conflictGroup 为 null 表示不属于任何冲突组

--- 满赠券说明 ---

满赠券不影响金额计算，但如果满足门槛条件，需要在计算结果中记录该券被使用。

--- 1. 理解与诊断 ---

  - 阅读现有代码，理解实现逻辑
  - 运行 mvn test，分析测试失败的原因
  - 不仅关注表面 bug，也关注代码架构层面的问题

--- 2. 架构重构 ---

  - 当前实现存在明显的架构问题，请重新设计并重构
  - 重构后的架构应易于扩展新的优惠券类型、修改叠加规则
  - 金额计算需要使用合适的数据类型，避免精度问题
  - 约束：CouponCalculator 接口签名不可修改

--- 3. Bug 修复与边界处理 ---

  - 修复所有导致测试失败的 bug
  - 异常输入不应导致程序崩溃（null、空列表、过期券、门槛不满足等）

--- 4. 计算明细 ---

扩展 CalculationResult，使其包含详细的计算过程信息：

  - 每张被使用的优惠券的优惠明细（券信息、优惠金额、应用顺序）
  - 未使用的优惠券列表及未使用原因（互斥排除、门槛不满足、已过期、冲突组排除等）

具体的数据结构由你自行设计。

--- 5. 优惠分摊（可选，完成以上内容后再做） ---

[此功能为加分项] 请在完成前述所有任务（架构重构、Bug 修复、计算明细、测试通过）后，
根据剩余时间决定是否实现。

计算完优惠后，需要将每张优惠券的优惠金额按比例分摊到参与优惠的商品行，用于退款和财务对账。

分摊规则：

  1. 按商品行金额占比分摊：金额 = 单价 x 数量
  2. 品类券范围限定：品类券只分摊到对应品类的商品行
  3. 精度要求：分摊金额精确到分（保留两位小数，HALF_UP 舍入）
  4. 尾差处理：分摊后各行加和可能与优惠总额有 +/-0.01 的误差，
     将尾差归入金额最大的商品行，保证「各行分摊金额之和 = 该券优惠总额（精确相等）」
  5. 最低实付保护：单行分摊后的实付金额不得低于 0.01 元
  6. 多券独立分摊：多张券各自独立分摊，按叠加顺序（折扣 -> 满减 -> 立减）

数据结构由你自行设计，以下为输出示例：

  示例：订单含 SKU-A（300元x1）和 SKU-B（200元x1），使用满200减50
  分摊结果：
    SKU-A: 分摊 30.00 元（300/500 x 50 = 30.00）
    SKU-B: 分摊 20.00 元（200/500 x 50 = 20.00）
    合计: 50.00 = 50.00

  尾差示例：订单含 SKU-A/B/C 各100元，立减10
    按比例: 3.33 + 3.33 + 3.33 = 9.99（差0.01）
    尾差归入金额最大行: 3.33 + 3.33 + 3.34 = 10.00
